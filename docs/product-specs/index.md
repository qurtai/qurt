## How To Use

- Each spec should define user problem, outcomes, and measurable success.
- Keep scope explicit: MVP first, then staged expansion.
- Link implementation plans in `docs/exec-plans/active/`.

## Spec Template

- Problem and user segment
- Goals and non-goals
- User journey
- Functional requirements
- UX and trust requirements
- Success metrics
- Risks and rollout plan
